# Advanced API Mediation Layer Configuration

Review the various use cases presented in this article, and the roles that perform them, and follow the links to the corresponding documentation that describes how to perform your specific customization. * [Single Sign On for Extending Services - JWT Token Configuration](./api-mediation/configuration-extender-jwt)
* [Signel Sign On for Extending Services - Passtickets Configuration](./api-mediation/configuration-extender-passtickets)
* [Customize management of API ML load limits (single instances and HA)](./api-mediation/configuration-customizing-management-of-apiml-load-limits)
* [Configure Unique cookie name for specific API ML instance](./api-mediation/configuration-unique-cookie-name-for-multiple-zowe-instances)
* [Customize the API Catalog UI](./api-mediation/configuration-customizing-the-api-catalog-ui)
* [Configure SAF Resource Checking](./api-mediation/configuration-saf-resource-checking)
* [Retreive a specific service within your environment](./api-mediation/configuration-access-specific-instance-of-service)
* [Set a consistent service ID](./api-mediation/configuration-set-consistent-service-id)

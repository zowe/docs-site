# Version 3.4.1 (January 2026)

Welcome to the Zowe Version 3.4.1 release!

See [New features and enhancements](#new-features-and-enhancements) for a full list of changes to the functionality. See [Bug fixes](#bug-fixes) for a list of issues addressed in this release.

**Download v3.4.1 build**: Want to try new features as soon as possible? You can download the v3.4.1 build from [Zowe.org](https://www.zowe.org/download.html).

## New features and enhancements

Zowe Version 3.4.1 contains the enhancement described in the following topic:

### Zowe API Mediation Layer

* API ML now supports binding on multiple network interfaces. ([#4457](https://github.com/zowe/api-layer/issues/4457))

## Bug fixes

Zowe Version 3.4.1 contains the bug fixes that are described in the following topics:

### Zowe API Mediation Layer

* Fixed infinispan configuration, exception handling, and the whole cache operation (v3) ([#4450](https://github.com/zowe/api-layer/issues/4450))
* Specified the expect 401 on the info endpoint. ([#4460](https://github.com/zowe/api-layer/issues/4460))
* Disabled hostname validation. ([#4452](https://github.com/zowe/api-layer/issues/4452))
* Fixed the error message during shutdown of the Discovery service. ([#4390](https://github.com/zowe/api-layer/issues/4390))
* Enabled loading jwks with nonstrict certificate validation. ([#4442](https://github.com/zowe/api-layer/issues/4442))
* Remove the reversecategorize filter from AT-TLS. ([#4441](https://github.com/zowe/api-layer/issues/4441))
* Applied AT-TLS in single service mode fixes. ([#4435](https://github.com/zowe/api-layer/issues/4435))
* Fixed Node.js enabler ([#4434](https://github.com/zowe/api-layer/issues/4434))
* Enabled loading JWKs from z/OSMF. ([#4432](https://github.com/zowe/api-layer/issues/4432))

### Vulnerabilities fixed

Zowe discloses fixed vulnerabilities in a timely manner giving you sufficient time to plan your upgrades. Zowe does not disclose the vulnerabilities fixed in the latest release as we respect the need for at least 45 days to decide when and how you upgrade Zowe. When a new release is published, Zowe publishes the vulnerabilities fixed in the previous release. For more information about the Zowe security policy, see the [Security page](https://www.zowe.org/security.html) on the Zowe website.

No security fixes were applied to this minor release.

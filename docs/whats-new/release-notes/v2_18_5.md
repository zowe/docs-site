# Version 2.18.5 (January 2026)

Welcome to the Zowe Version 2.18.5 release!

See [Bug fixes](#bug-fixes) for a list of issues addressed in this release.

**Download v2.18.5 build**: Want to try new features as soon as possible? You can download the v2.18.5 build from [Zowe.org](https://www.zowe.org/download.html).

## New features and enhancements

Zowe Version 2.18.5 contains the enhancements that are described in the following topics.

### Zowe API Mediation Layer

- Implemented allowlist for URL domains (v2). This feature validates all service registration metadata against a trusted domain allowlist and issues a warning for for any domains that are not included in the allowlist. The system automatically trusts `zowe.externalDomains` (in a both single instance and HA setup), `zowe.haInstances.<id>.hostname` (in an HA setup), the target hostname defined and trusted corporate and vendor infrastructure domain. ([#4689](https://github.com/zowe/api-layer/issues/4689))

## Bug fixes

Zowe Version 2.18.5 contains the bug fixes that are described in the following topics.

### Zowe API Mediation Layer

* Restricted caching to items originating exclusively from trusted proxies. ([#4706](https://github.com/zowe/api-layer/issues/4706))
* Fixed Server-Sent Events (SSE) data corruption issues. ([#4530](https://github.com/zowe/api-layer/issues/4530))
* Resolved an issue preventing all headers from being written in HTTP responses. ([#4522](https://github.com/zowe/api-layer/issues/4522))
* Downgraded the domain allowlist enforcement to issue warnings by default in version 2.18.5. ([#4733](https://github.com/zowe/api-layer/issues/4733))
* Applied relaxed firewall rules strictly to routed requests (v2). ([#4666](https://github.com/zowe/api-layer/issues/4666))
* Standardized the host verification scheme to align with the rest of the Gateway. ([#4667](https://github.com/zowe/api-layer/issues/4667))

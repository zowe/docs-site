Write the documentation with accessibility in mind. This is also a feedback in the Zowe community survey: `more attention to accessibility, current docs are almost unusable for visually impaired users. Spend some time with it blindfolded at a Mac with VoiceOver enabled.`

To do that, need to do the following:
- Consolidate general accessibility guidelines and examples. 
- Add required guidelines to the Zowe style guide. 
- Walk through the existing documentation to catch and fix accessibility issues. 
- Ensure that new docs are written with accessibility in mind. 
- Investigste tools for checking accessibility
